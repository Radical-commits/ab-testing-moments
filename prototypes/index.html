A/B Testing for Broadcast Campaigns

Interactive UI prototypes showing 4 different approaches for adding A/B testing to the Moments broadcast campaign creation flow.

4 prototypes Phase 1 scope February 2026

What These Prototypes Cover

  • Full campaign creation flow (channel, recipients, message, settings, launch)
  • Two content variants (A and B) per campaign
  • Two test modes: Equal Split (50/50) and Test-Then-Send-Winner
  • Winner criteria: open rate, click rate, delivery rate
  • Test group size slider (10% - 50%)
  • Wait period configuration (1h, 4h, 12h, 24h, 48h)
  • Side-by-side results analytics with bar charts
  • Confidence labels: Confident, Promising, Not enough data
  • "Copy from A" button for quick variant creation
  • Mock data: Variant B wins with 51.7% open rate vs 45.2%
Prototypes
1

A/B Toggle

Minimal Disruption

A simple toggle inside the Message Composition step enables A/B testing. When turned on, the editor splits into two variant tabs and a config panel appears below. The rest of the broadcast flow stays the same. A/B testing is treated as an optional add-on.

+ Easy to build + Minimal learning curve + No extra steps for non-test campaigns - A/B feels like an afterthought - Config buried in message step

2

Dedicated Step

A/B as a First-Class Step

A new "Test Setup" step is added between Recipients and Message Composition. Users choose between "Send to Everyone" or "A/B Test," then configure the test settings. Message Composition then shows side-by-side editors for both variants. A/B testing has its own place in the flow.

+ Makes A/B testing visible and structured + Side-by-side editing is clear + Config separated from content - Adds a step to every campaign - Extra click for non-test campaigns
Open Prototype →
3

Branch from Creation

Choose Campaign Type Upfront

Before entering the broadcast flow, users choose between "Standard Broadcast" and "A/B Test Broadcast." The A/B path enters a tailored flow with variant editors and test configuration built in. Standard campaigns remain completely unchanged.

+ Clean separation of flows + Standard campaigns stay simple + A/B feels like a full feature - Must decide upfront before starting - Duplicates some flow logic - Cannot switch mid-flow
Open Prototype →
4

Progressive Disclosure

"Add Variant B" Inline

After writing the main message (Variant A), a prominent "Add Variant B for A/B Testing" card appears below the editor. Clicking it expands inline to show the Variant B editor and test configuration. Users write their message first, then decide to test. The variant can be removed just as easily.

+ Natural: compose first, test second + Discoverable without being forced + Easily reversible + No upfront decisions needed - Message step gets long with A/B enabled - Config and content in same step
