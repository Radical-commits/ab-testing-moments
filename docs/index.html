01

Approaches

01
☰
A/B Toggle
Minimal Disruption

A simple toggle inside the Message Composition step enables A/B testing. When turned on, the editor splits into two variant tabs and a config panel appears below. The rest of the broadcast flow stays the same.

Easy to build Minimal learning curve No extra steps Feels like add-on Config buried in step
Open Prototype →
02
▦
Dedicated Step
A/B as a First-Class Step

A new "Test Setup" step is added between Recipients and Message Composition. Users choose between "Send to Everyone" or "A/B Test," then configure settings. Message step shows side-by-side editors for both variants.

Visible and structured Side-by-side editing Config separated Adds step to every campaign Extra click for non-tests
Open Prototype →
03
★
Branch from Creation
Choose Campaign Type Upfront

Before entering the broadcast flow, users choose between "Standard Broadcast" and "A/B Test Broadcast." The A/B path enters a tailored flow with variant editors and test configuration built in. Standard campaigns remain unchanged.

Clean separation Standard stays simple Feels like full feature Must decide upfront Duplicates flow logic Cannot switch mid-flow
Open Prototype →
04
✚
Progressive Disclosure
"Add Variant B" Inline

After writing the main message, a prominent "Add Variant B for A/B Testing" card appears below the editor. Clicking it expands inline to show the B editor and test config. Users write first, then decide to test. Easily reversible.

Compose first, test second Discoverable not forced Easily reversible No upfront decisions Message step gets long Config mixed with content
