A/B Testing for
Broadcast Campaigns

Four approaches for adding A/B testing to the broadcast creation flow. Each prototype is interactive.

01

A/B Toggle

Minimal Disruption

A toggle inside the Message Composition step enables A/B testing. The editor splits into two variant tabs and a config panel appears below. The rest of the flow stays the same.

Easy to build Low learning curve No extra steps Feels like add-on Config buried in step
Open prototype →
02

Dedicated Step

A/B as a First-Class Step

A new "Test Setup" step between Recipients and Message Composition. Users choose "Send to Everyone" or "A/B Test," then configure settings. Message step shows side-by-side editors.

Visible and structured Side-by-side editing Config separated Adds step to every campaign Extra click for non-tests
Open prototype →
03

Branch from Creation

Choose Campaign Type Upfront

Before entering the flow, users choose "Standard Broadcast" or "A/B Test Broadcast." The A/B path has variant editors and test configuration built in. Standard campaigns stay unchanged.

Clean separation Standard stays simple Feels like full feature Must decide upfront Duplicates flow logic Cannot switch mid-flow
Open prototype →
04

Progressive Disclosure

"Add Variant B" Inline

After writing the main message, an "Add Variant B" card appears below the editor. Clicking it expands inline to show the B editor and test config. Write first, then decide to test.

Compose first, test second Discoverable not forced Easily reversible Message step gets long Config mixed with content
Open prototype →
